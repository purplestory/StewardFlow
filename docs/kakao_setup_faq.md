# 카카오 로그인 설정 FAQ

## Q: Supabase 설정을 변경한 후 서버를 재시작해야 하나요?

### A: 아니요, 재시작할 필요 없습니다

**Supabase 설정 변경은 즉시 반영됩니다:**
- Supabase 설정은 서버 측 설정이므로 Next.js 개발 서버를 재시작할 필요가 없습니다
- 설정을 저장하면 바로 적용됩니다

**하지만 다음은 확인하세요:**

1. **브라우저 새로고침**
   - Supabase 대시보드에서 설정을 변경한 후
   - 브라우저를 새로고침 (F5 또는 Ctrl+R)
   - 설정이 제대로 저장되었는지 확인

2. **캐시 클리어 (필요시)**
   - 브라우저 캐시 문제가 있을 수 있습니다
   - 강력 새로고침: Ctrl+Shift+R (Windows) 또는 Cmd+Shift+R (Mac)

3. **Supabase 설정 저장 확인**
   - "Save" 버튼을 클릭했는지 확인
   - 저장 완료 메시지가 표시되었는지 확인

## Q: 언제 서버를 재시작해야 하나요?

### A: 다음 경우에만 재시작이 필요합니다:

1. **환경 변수 변경**
   - `.env.local` 파일을 수정한 경우
   - `NEXT_PUBLIC_SUPABASE_URL` 또는 `NEXT_PUBLIC_SUPABASE_ANON_KEY` 변경

2. **코드 변경**
   - `src/` 폴더의 코드를 수정한 경우
   - 컴포넌트나 로직을 변경한 경우

3. **패키지 설치**
   - `npm install`로 새 패키지를 설치한 경우

### Supabase 설정 변경은 재시작 불필요:
- ✅ Authentication → Providers 설정 변경
- ✅ Kakao Provider 활성화/비활성화
- ✅ Client ID, Client Secret 입력
- ✅ Redirect URL 설정

## Q: 설정이 반영되지 않을 때는?

### 해결 방법:

1. **Supabase 대시보드 확인**
   - 설정이 제대로 저장되었는지 확인
   - "Enable Kakao" 토글이 ON인지 확인

2. **브라우저 새로고침**
   - Supabase 대시보드 새로고침
   - 앱 페이지 새로고침

3. **캐시 클리어**
   - 브라우저 캐시 삭제
   - 시크릿 모드에서 테스트

4. **잠시 대기**
   - Supabase 설정 변경이 전파되는데 몇 초 걸릴 수 있습니다
   - 10-30초 정도 기다린 후 다시 시도

## 요약

| 변경 사항 | 서버 재시작 필요? |
|---------|----------------|
| Supabase 설정 변경 | ❌ 불필요 |
| 환경 변수 변경 | ✅ 필요 |
| 코드 변경 | ✅ 필요 |
| 패키지 설치 | ✅ 필요 |

**결론**: Supabase 설정 변경 후에는 서버를 재시작할 필요 없이, 브라우저만 새로고침하면 됩니다.
