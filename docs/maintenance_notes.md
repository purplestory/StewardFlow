# 유지보수 메모

본 문서는 제작/운영 중 반복적으로 확인해야 하는 항목을 정리합니다.

## 1) 데이터/보안
- RLS 활성화 및 정책 변경 시 테스트
- 조직 ID 필드가 모든 테이블에 적용되는지 확인
- 관리자/부서관리자 권한 변화 시 감사 로그 확인

## 2) 운영 점검
- 초대 만료/취소 처리 확인
- 승인 정책 변경 후 실제 예약 흐름 점검
- 알림 생성 여부 및 읽음 처리 확인
- 감사 로그 필터 정상 동작 확인

## 3) 감가상각/자산 수명 (기초 데이터)
- `purchase_date`, `purchase_price`, `useful_life_years` 입력률 점검
- 마지막 사용일(`last_used_at`) 업데이트 확인
- 장기 미사용 자산 리스트(향후 리포트 기준)

## 4) 장애 대응
- 예약 상태 변경 API 실패 시 메시지 확인
- Storage 업로드 실패 시 재시도/메시지 확인

## 5) 다음 단계 후보
- 감가상각 리포트(직선법)
- 장기 미사용 자산 리포트
- 행사/절기 자산 태그와 필터
